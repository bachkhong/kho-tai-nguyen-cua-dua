✔️ Checklist học thật, không học ảo

Sau mỗi bài SQL phải có ít nhất 3 query tự viết lại không nhìn đáp án.
Sau mỗi dashboard phải có một trang "logic metric" giải thích cách tính.
Sau mỗi project phải có README: bối cảnh, dữ liệu, câu hỏi, công cụ, kết quả, giới hạn.

Mỗi tháng phải review lại project cũ và sửa ít nhất 1 điểm: query, model, chart, narrative hoặc tài liệu.
Kỹ năng

Knowledge Map theo công cụ

Những gì bạn cần nắm vững ở từng mảng kỹ năng

📊

Excel

  • Bảng dữ liệu sạch: mỗi cột một biến, mỗi dòng một quan sát
  • Hàm: SUMIFS, COUNTIFS, XLOOKUP, INDEX/MATCH, FILTER, SORT, UNIQUE
  • PivotTable/PivotChart: summarize, grouping time, slicer, timeline
  • Power Query: import, append, merge, unpivot, trim/clean
  • Dashboard: KPI card, trend line, breakdown, filter, action note
🔢

Google Sheets

  • Hàm: QUERY, FILTER, ARRAYFORMULA, IMPORTRANGE, REGEXMATCH
  • Pivot table, chart, conditional formatting, data validation
  • Automation: Apps Script, time trigger, email/report automation
  • Kết nối Looker Studio để làm dashboard nhẹ cho marketing/content
🗄️

SQL Server / T-SQL

  • Query nền: SELECT, WHERE, ORDER BY, DISTINCT, aliases
  • Tổng hợp: GROUP BY, HAVING, aggregate functions, date bucket
  • JOIN đúng: inner, left, anti-join, tránh nhân dòng
  • Logic: CASE WHEN, CTE, subquery, window functions, cohort
  • Đi làm: schema, data dict, stored procedures, index, production safety
📈

Power BI

  • Power Query: làm sạch trước model, đặt tên query dễ hiểu
  • Data model: fact/dimension, star schema, date table
  • DAX: CALCULATE, FILTER, ALL, time intelligence, iterator
  • Visualization: KPI first, trend, comparison, drill-down
  • Service: publish, refresh, gateway, row-level security
🐍

Python for Data

  • Python basics: variable, list, dict, loop, function
  • Pandas: read, clean, filter, groupby, merge, pivot, datetime
  • EDA: distribution, outlier, correlation, trend, segment, RFM
  • Visualization: matplotlib/seaborn, chart choice, annotation
  • Automation: batch clean, API, refresh dataset, generate report
📖

Storytelling with Data

  • Luôn bắt đầu từ audience: ai đọc, họ cần quyết định gì
  • Thông điệp trước biểu đồ: mỗi chart trả lời một câu hỏi rõ
  • Chọn chart theo mục đích: so sánh, xu hướng, cơ cấu
  • Giảm nhiễu: bỏ gridline thừa, màu không cần thiết
  • Kết luận: phát hiện → số liệu → nguyên nhân → đề xuất
💼

Workplace & Senior Skills

  • Clarify requirement: metric definition, time range, filter, owner
  • Data quality: missing, duplicate, outlier, grain, join cardinality
  • Production DB safety: read-only, limit/sampling, hỏi quyền
  • Documentation: data dictionary, logic measure, changelog
  • Stakeholder: nói bằng business impact, không chỉ bằng tool

🎙 Mock Interview Script theo vòng

Round 1 — SQL Fundamentals

  • Explain INNER JOIN vs LEFT JOIN.
  • What can cause duplicated rows after JOIN?
  • How do you calculate revenue when orders have status?
  • Write a query for monthly active users.

Round 2 — Business Case

  • Revenue dropped this month. What dimensions do you check first?
  • How would you separate volume effect and price effect?
  • How would you decide whether a campaign is successful?
  • What is a bad insight? Give an example.

Round 3 — Dashboard & BI

  • What makes a dashboard useful?
  • When should you use Power Query vs DAX?
  • How do you validate a Power BI number?
  • What is a star schema and why does it matter?

Round 4 — Python & EDA

  • How do you handle missing values?
  • How do you detect outliers?
  • When would you use Python instead of Excel?
  • How do you make your notebook readable for reviewers?

Round 5 — Senior Behavior

  • Tell me about a time data was incomplete.
  • How do you push back when stakeholder asks for a vague report?
  • How do you document metric logic?
  • How do you prioritize multiple data requests?

Checklist Data Analyst thực chiến

Những điều cần làm trong từng tình huống công việc thực tế

🗣 Khi nhận request từ stakeholder

Câu hỏi kinh doanh là gì? Người xem sẽ quyết định gì?
Metric cần tính là gì? Công thức, filter, time range, source?
Cần so sánh với mốc nào: MoM, YoY, target, budget?
Dữ liệu nằm ở đâu? Owner là ai? Có data dictionary không?
Output mong muốn: query, bảng số, dashboard, slide hay recommendation?
Deadline và mức độ chính xác cần thiết là gì?

🖥 Khi làm với production database

Ưu tiên read-only; không UPDATE/DELETE khi chưa được giao quyền.
Luôn SELECT TOP/LIMIT trước để hiểu dữ liệu.
Kiểm tra grain của bảng trước khi JOIN.
Không chạy query nặng giờ cao điểm nếu chưa biết impact.
Không export dữ liệu nhạy cảm ra file cá nhân.
Lưu query version và ghi chú ngày chạy, source, filter.

📊 Khi validate dashboard

KPI chính phải đối chiếu được bằng SQL hoặc source report.
Kiểm tra filter/slicer có làm thay đổi số đúng kỳ vọng không.
Kiểm tra model có gây double counting không.
Kiểm tra date table, active relationship, direction filter.
Kiểm tra refresh date và dữ liệu mới nhất.
Viết known caveats: số nào chưa chắc, nguồn nào thiếu.

💡 Khi viết insight

Insight phải trả lời "vậy thì sao?", không chỉ mô tả chart.
Luôn có số liệu: mức tăng/giảm, tỷ trọng, nhóm tác động chính.
Nêu nguyên nhân khả nghi, không khẳng định quá mức.
Đề xuất hành động: kiểm tra, tối ưu, thử nghiệm, thay đổi quy trình.
Ghi giới hạn: thiếu dữ liệu nào, cần nguồn nào để kết luận chắc hơn.
